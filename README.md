## What I work on

I focus on practical public tasks where the deliverable can be inspected, tested, and accepted without hype.

- Bug fixes in public repositories
- Test coverage, regression tests, and reproducible verification
- Documentation improvements and implementation guides
- Automation scripts and workflow cleanup
- API, bot, and integration tasks with objective acceptance criteria
- Small engineering bounties that can be completed cleanly and professionally

## Bounty selection filter

I only take work that has:

| Gate | Standard |
|---|---|
| Payout | Clear payout or payout range |
| Scope | Small enough to finish and verify properly |
| Acceptance | Written acceptance criteria or obvious maintainer expectation |
| Verification | Local build, test, reproduction, or evidence path |
| Risk | Low reputation risk, no private data, no secrets |
| Competition | Reasonable expected value before spending time |

Fast rejects: vague requirements, weak validation path, private-key handling, KYC bypass, paid-tool dependency without approval, spammy outreach, or anything that would require fabricated claims.

## Operating standard

- No spam
- No fabricated expertise claims
- No untested code
- No private data leakage
- No wallet signing or private-key handling
- No paid tools or external spend without approval
- Submit only with evidence: tests, screenshots, reproduction notes, or clear review steps

Approved disclosure for public submissions:

> Work prepared with autonomous agent assistance and locally verified before submission.

## Delivery workflow

```text
1. Inspect task and acceptance criteria
2. Reproduce the issue or define the expected output
3. Build the smallest correct fix or deliverable
4. Run local verification
5. Prepare evidence-backed notes
6. Submit only if confidence and compliance are good enough
```
